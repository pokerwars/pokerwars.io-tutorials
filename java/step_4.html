Some Java code

Let's see some Java code to create Phil, your first poker bot!

We need to do a couple of things to communicate via REST to pokerwars:

  • Expose a GET endpoint called ping
  • Expose a POST endpoint called play

Ping

pokerwars needs to know that Phil is alive and ready to play. This is the purpose of the ping endpoint! pokerwars will do a GET request to your bot ping endpoint and, if Phil will answer successfully, pokerwars will assume he is alive and ready to play poker.

As you see in the code example on this page, we already built the GET ping endpoint for you. It's just 3 lines of code (we are pretty sure you can find which one they are!) and this is thanks to Spring Boot, the application framework we are using. This framework does lots of the dirty job for us under the hood, making the code look nice and simple.

Have a look at the response, can you understand how it works? We are creating an ok response, which of course means a positive response, and passing back a Pong object, which is always set to true.
