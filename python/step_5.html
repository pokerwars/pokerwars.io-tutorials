Theory in action

Now, finally, we can have fun. Let's start with creating one endpoint: play.

As we figured out before, the play endpoint is invoked by pokerwars.io system sending the game information and it's asking for an action as return so it's a typical POST case.

We already created the notifications endpoint. Starting with the bottle command to identify the type of endpoint @post followed by the URL that we want to expose. In this case the URL starts with the domain name /pokerwars.io and it's followed by the verb connected to the action. 

@post('/pokerwars.io/notifications')

Now it's your turn, You need to create your play endpoint following what we've learned so fa. Have fun!!!
